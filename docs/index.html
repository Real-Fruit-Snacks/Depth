FEATURES

Full SSH stack in assembly

Every cryptographic primitive, every protocol message, every byte — handwritten x86_64 NASM. No libc. No shortcuts.

ChaCha20-Poly1305 AEAD

Full chacha20-poly1305@openssh.com transport encryption. Two-key scheme: K1 encrypts payload (counter=1), K2 encrypts packet length (counter=0). Sequence-number nonce. Every packet authenticated — tampered data rejected before decryption.

K2 encrypts length field (counter=0, seq as nonce)
K1 encrypts padded payload (counter=1)
Poly1305 key from K1 block 0
MAC covers enc_length + enc_payload
94KB
Static ELF Binary

Ed25519 + X25519

Complete elliptic curve cryptography from scratch. X25519 Diffie-Hellman for key exchange, Ed25519 for host key signatures. SHA-512 internals, SHA-256 exchange hash. All field arithmetic in pure assembly.

Full SSH Protocol Stack

RFC 4253/4254 compliant: version exchange, algorithm negotiation (KEXINIT), ECDH key exchange, NEWKEYS, service request, password auth, channel multiplexing (8 concurrent).

Interactive PTY Shell

Full pseudoterminal support via /dev/ptmx. Fork, setsid, set controlling terminal, dup2 stdio, execve /bin/bash. Poll-based I/O relay with child lifecycle management.

SFTP File Transfers

Full SFTPv3 implementation: open, read, write, close, stat, fstat, lstat, setstat, opendir, readdir, remove, mkdir, rmdir, rename, realpath. Handles concurrent shell + SFTP sessions through event-loop integration.

$ sftp -P 7777 svc@target

sftp> ls -la /tmp/

Port Forwarding

Both local (ssh -L) and remote (ssh -R) TCP forwarding. Direct-tcpip channels for local forward, global request handling for remote forward. Multiplexed alongside shell/SFTP.

ARCHITECTURE

Four-layer design

Clean separation between cryptographic primitives, protocol state machine, channel multiplexing, and subsystem dispatch.

1

Crypto Primitives

X25519 scalar multiplication, Ed25519 sign/verify, ChaCha20 stream cipher, Poly1305 MAC, SHA-256, SHA-512, HMAC-SHA256, HKDF. All from-scratch in x86_64 assembly.

2

Protocol Layer

SSH version exchange, KEXINIT algorithm negotiation, ECDH key exchange with X25519, NEWKEYS transition, session key derivation (A-F letters via HKDF).

3

Channel Layer

Up to 8 multiplexed channels with independent window management. Session, direct-tcpip, and forwarded-tcpip channel types. PTY allocation, shell spawn, pipe exec.

4

Subsystems

SFTPv3 with 16 file opcodes and handle table. Local and remote TCP port forwarding. Poll-based event loop dispatching across all active channels and forwarded connections.

GETTING STARTED

Build from source

NASM + ld. That's it. No package manager, no build system beyond make.

build.sh
$ git clone https://github.com/Real-Fruit-Snacks/Depth.git
$ cd Depth && make

$ file build/depth
ELF 64-bit LSB executable, x86-64
statically linked, no section header

$ ls -la build/depth
-rwxr-xr-x 1 user user 94208 depth
usage.sh
# Start SSH server (bind mode)
$ ./build/depth

# Connect with OpenSSH
$ ssh -p 7777 svc@target

# SFTP file transfer
$ sftp -P 7777 svc@target

# Local port forwarding
$ ssh -L 8080:internal:80 -p 7777 svc@target
