Protocol Features

Full SSH-2.0 stack built from scratch in x86_64 assembly. Every layer implemented with pure Linux syscalls, no libc.

🔒

Key Exchange

X25519 Diffie-Hellman with ephemeral keys. SHA-256 exchange hash. RFC 4253 key derivation producing 6 session keys (A-F).

curve25519-sha256 + ssh-ed25519
🛡

Encrypted Transport

ChaCha20-Poly1305 AEAD. Two-key scheme: K2 encrypts length (ctr=0), K1 encrypts payload (ctr=1). Sequence-number nonce.

chacha20-poly1305@openssh.com
🖨

Interactive Shell

PTY via /dev/ptmx. Fork, setsid, set controlling terminal, dup2 stdio, execve. Poll-based I/O relay with child lifecycle management.

ssh -p 7777 svc@target
📦

SFTP Transfer

Full SFTPv3: open, read, write, close, stat, opendir, readdir, remove, mkdir, rmdir, rename, realpath. 16 concurrent handle slots.

sftp -P 7777 svc@target
🔀

Port Forwarding

Local (direct-tcpip) and remote (tcpip-forward) TCP forwarding. Multiplexed alongside shell and SFTP channels.

ssh -L 8080:internal:80 -p 7777 svc@target
⚡

Pipe Execution

Non-interactive commands without PTY. Creates stdin/stdout pipes, forks, executes via bash -c. Graceful EOF and drain.

ssh -p 7777 svc@target 'whoami'

Cryptography

Every cryptographic primitive implemented from scratch in x86_64 assembly. No libraries. No shortcuts.

ChaCha20

20-round stream cipher with 256-bit key. Two independent instances: K1 for payload encryption, K2 for length encryption. Counter-based with sequence nonce.

Poly1305

One-time authenticator with mod 2^130-5 arithmetic. Computes 16-byte MAC over encrypted length + payload. Key derived from ChaCha20 block 0.

X25519

Curve25519 scalar multiplication for Diffie-Hellman key exchange. Ephemeral keypairs per session. Montgomery ladder implementation.

Ed25519

Edwards-curve digital signatures for host key authentication. Sign and verify operations. SHA-512 for internal hashing. Complete field arithmetic.

SHA-256 / SHA-512

FIPS 180-4 compliant. SHA-256 for exchange hash and HKDF. SHA-512 for Ed25519 internals. Streaming interface for large inputs.

HMAC + HKDF

HMAC-SHA256 keyed hashing. HKDF extract-and-expand (RFC 5869) for session key derivation. Produces 6 keys per RFC 4253 (A-F).

Architecture

Layered protocol stack. Each layer implemented as independent assembly modules with clean interfaces.

1

Network Layer

Raw TCP via Linux syscalls: socket, connect, bind, listen, accept. Function-pointer I/O dispatch enables transparent TLS wrapping.

2

Crypto Layer

8 primitives from scratch: ChaCha20, Poly1305, X25519, Ed25519, SHA-256, SHA-512, HMAC-SHA256, HKDF. All field arithmetic in pure assembly.

3

SSH Protocol

RFC 4253/4254 compliant. Version exchange, KEXINIT negotiation, ECDH key exchange, NEWKEYS transition, service requests, authentication, channel operations.

4

Application Layer

PTY shell, pipe exec, SFTPv3, TCP forwarding. Poll-based event loop with 13-entry pollfd array. Up to 8 multiplexed channels with independent windows.

Getting Started

Build from source with NASM and GNU ld. Linux x86_64 only.

build.sh 
# Clone and build
$ git clone https://github.com/Real-Fruit-Snacks/Depth.git
$ cd Depth

# Build binary + test harnesses
$ make

# Run 268 tests
$ make test
268 passed in 12.4s

# Verify binary
$ file build/depth
ELF 64-bit LSB executable, x86-64, statically linked
$ ls -la build/depth
-rwxr-xr-x 1 user user 94208 depth
usage.sh 
# Start SSH server (bind mode)
target$ ./build/depth

# Connect with OpenSSH
attacker$ ssh -p 7777 svc@target
svc@target:~$

# Remote command execution
$ ssh -p 7777 svc@target 'id'
uid=1000(svc) gid=1000(svc)

# SFTP file transfer
$ sftp -P 7777 svc@target
sftp> put payload.bin /tmp/

# Port forwarding
$ ssh -L 8080:172.16.1.5:80 -p 7777 svc@target
